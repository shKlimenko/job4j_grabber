new User(100, "Ivan", "5553235")


Объект класса User будет занимать: 136 байта (
                            12b заголовок,
                            4b поле int,
                            4 поле String + 56b объект "Ivan",
                            4 поле String + 56b объект "5553235")


"ivan"
Заголовок 					= 12b
String поля:byte			= 1b
		int				    = 4b
		boolean			    = 4b (размер типа boolean, зависит от реализации конкретной виртуальной машины, в основном 4b)
		byte[]			    = 4b (ссылочная переменная на объект массива)
		выравнивание		= 7b до 32b

		new byte[4]			= 16b (12b заголовок + 4b на длину массива)
		примитивы byte		= 4b (1byte * 4)
		выравнивание		= 4b до 24b

Итого 56b (32b + 24b)


"5553235"
Заголовок 					= 12b
String поля:byte			= 1b
		int				    = 4b
		boolean			    = 4b
		byte[]			    = 4b (ссылочная переменная на объект массива)
		выравнивание		= 7b до 32b

		new byte[7]			= 16b (12b заголовок + 4b на длину массива)
		примитивы byte		= 7b (1byte * 7)
		выравнивание		= 1b до 24b

Итого 56b (32b + 24b)


В программе мы создаем 10000 объектов. Под каждый объект происходит выделение памяти.
Все объекты рождаются в eden (young generation). Когда заполняется eden происходит minor GC, объекты у которых нет ссылки удаляются.
Остальные попадают в survivor space. При перемещении объектов растет их возраст.
Объекты, которые достигли определенного возраста попадают в old generation.
Когда old generation заполняется производиться major GC.
